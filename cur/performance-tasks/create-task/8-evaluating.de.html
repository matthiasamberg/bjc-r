Deine Arbeit evaluieren

TG: Rubric austeilen.

Auf dieser Seite überprüfst du alle Create-Task-Komponenten, bereitest dich weiter auf den schriftlichen Teil vor und feierst deinen Erfolg!

Während der Übungs-Create-Task kannst du noch mit Lehrkraft und Mitschüler*innen sprechen. Bei der offiziellen Create Task erhältst du keinerlei Hilfe von der Lehrkraft, und dein*e Partner*in darf nur beim Programmieren unterstützen.
  1. Wenn Programm, Video, Personalized Project Reference und Übungsschreiben fertig sind, vergleiche alles sorgfältig mit den Anforderungen in den AP Student Handouts und den Rubrics für den schriftlichen Teil.
    Hoffe nicht einfach auf Punkte – achte auf Details: Bist du dir wirklich sicher, dass du für jeden einzelnen Teil volle Punktzahl bekommst? Nimm Anpassungen vor, bis du davon überzeugt bist.
  2. Wenn du an der Übungs-Create-Task arbeitest, bitte eine Klassenkameradin/einen Klassenkameraden und/oder deine Lehrkraft um Feedback. Falls dich jemand um Review bittet, sei ehrlich und detailgenau – hilfreiches Feedback darf freundlich, aber muss wahr sein.
  3. Zwischen der Abgabe von Programm, Video und Reference Sheet und dem schriftlichen AP-Teil liegt etwas Zeit. Nutze sie: Übe weitere Beispielaufgaben aus veröffentlichten Examens. Du solltest Fragen zu deinem Programm beantworten können, z. B.:
    • Beschreibe den Gesamtzweck des Programms.
    • Beschreibe die Funktionalität, die dein Video zeigt.
    • Beschreibe Ein- und Ausgaben, die im Video vorkommen.
    • Erkläre, welche Daten in der Liste auf deinem Personalized Project Reference stecken.
    • Erkläre, wie die Liste Komplexität reduziert, indem du beschreibst, warum dein Code ohne die Liste nicht (oder nur deutlich komplizierter) funktionieren würde.
    • Beschreibe die Prozedur auf deinem Personalized Project Reference und wie sie zur Gesamtfunktion beiträgt.
    • Erkläre detailliert, wie der Algorithmus in dieser Prozedur funktioniert – so ausführlich, dass jemand ihn nachbauen kann.
    • Beschreibe, welche Bedingung geprüft wird, wenn deine Prozedur aufgerufen wird (durch den Code auf deinem Reference Sheet) und welches Ergebnis dieser Aufruf hat.
  4. Feiere deinen Abschluss! Ein AP-Kurs erfordert viel Einsatz – gönn dir etwas Besonderes als Belohnung.
