Dein persönliches Projekt-Referenzblatt vorbereiten

Auf dieser Seite erstellst du auf Basis deines Snap!-Projekts ein Personalized Project Reference Sheet für die AP-Prüfung.

Es gibt zwei Stellen, an denen du Codebilder an das College Board übermittelst:

  • Für Komponente A (Programmiercode) reichst du ein Bild deines gesamten Codes ein – inklusive Kommentare und Quellenangaben.
  • Für Komponente C (Personalized Project Reference) reichst du Bilder von konkreten Ausschnitten ein – ohne Kommentare.
Während des schriftlichen Prüfungsteils darfst du nur auf deinem Personalized Project Reference Sheet nachschauen.

  1. Befolge die College-Board-Vorgaben und markiere in deinem Projekt zwei Codeausschnitte zu Prozeduren und zwei Ausschnitte zu Listen. Hier eine kurze Erinnerung – halte dich immer an die offiziellen Anforderungen:
    1. Die zwei Prozedur-Ausschnitte müssen zeigen:
      1. Eine selbst entwickelte Prozedur mit Name, ggf. Rückgabetyp (Zahl, Text, Liste …), Eingaben und deren Verwendung sowie dem Algorithmus im Inneren, der Sequenz, Auswahl und Iteration enthält.
      2. Einen Abschnitt deines Programms, der diese Prozedur aufruft.
    2. Die zwei Listen-Ausschnitte müssen zeigen:
      1. Einen Abschnitt, in dem Daten in einer Liste gespeichert werden, inkl. des Listennamens.
      2. Einen Abschnitt, in dem genau diese Liste für den Programmzweck genutzt wird.
  2. Erstelle dein Personalized Project Reference Sheet mit genau diesen vier Ausschnitten.
    In der AP-Prüfung bekommst du einen Ausdruck deines Personalized Project Reference Sheets. In der Übungs-Create-Task darfst du es überarbeiten, falls du die Schreibaufgaben auf der nächsten Seite damit nicht beantworten kannst. In der echten Prüfung darfst du es nicht mehr ändern – nutze daher die Übung, um zu überlegen, welche Codeausschnitte sich eignen. Wenn du das Referenzblatt für die offizielle Create Task erstellst, denke vorher darüber nach, wie gut du damit die Fragen auf der nächsten Seite beantworten kannst.

    3. Code als Bild exportieren

    Snap! bietet zwei praktische Wege, Bilddateien deines Codes zu erzeugen:

    • Script Pics erstellen

      Du kannst jedes Skript/Block als PNG speichern, indem du ihn rechtsklickst (Mac: Ctrl-Klick) und „script pic…“ wählst. Die Datei landet im Downloads-Ordner.

      Wenn du einen Reporter-Block mit gedrückter Shift-Taste rechtsklickst (Mac: Shift-Ctrl-Klick), enthält das Script Pic die Ausgabeblase.

    • Alle Bilder auf einmal exportieren

      Du kannst eine HTML-Datei mit allen Blöcken deines Projekts sowie einem Bühnenbild exportieren, indem du im Snap!-Dateimenü „Export summary…“ wählst (). Die Datei landet ebenfalls im Downloads-Ordner. Öffne sie (im Browser), klicke die gewünschten Bilder an (Rechtsklick bzw. Mac: Ctrl-Klick), kopiere sie und füge sie in dein Dokument ein.
