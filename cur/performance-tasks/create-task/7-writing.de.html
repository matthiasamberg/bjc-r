Über dein Projekt schreiben

Auf dieser Seite übst du das Schreiben über dein Projekt zur Vorbereitung auf die AP-Prüfung.

Der schriftliche Teil der Create Task findet während der AP-Prüfung nach den Multiple-Choice-Fragen statt. Du darfst dabei nur das vom College Board bereitgestellte Referenzblatt und dein Personalized Project Reference Sheet (Komponente C) verwenden.

Es gibt vier Fragen – jeweils zu einem dieser Bereiche:

  • Design, Funktion und Zweck deines Programms – für wen ist es gedacht, was tut es, wie funktioniert es, wie hast du es gebaut, welche Ein- und Ausgaben gibt es?
  • Algorithmische Entwicklung – wie funktionieren deine Algorithmen (inkl. Operatoren/Conditionals)? Was passiert, wenn du sie veränderst?
  • Fehler und Tests – welche Fehler könnten auftreten, wie würdest du darauf testen oder sie beheben? (Siehe Seite 4 „Testen“ für Begriffe/Techniken.)
  • Daten- und Prozedurabstraktion – wie helfen Listen und/oder Prozeduren dabei, Komplexität zu managen (inkl. Fragen zu Datenabstraktion oder Listendurchlauf)?

Die tatsächlichen Fragen in der Prüfung variieren. Hier findest du Beispielfragen des College Board, passend zu den vier Bereichen oben.

  1. Bestimme eine Zielperson für dein Programm. Beschreibe, wie das Design deines Programms deren Bedürfnisse erfüllt.
  2. Betrachte die erste Schleife im Abschnitt „Procedure“ deines Personalized Project Reference Sheets. Bestimme, wie oft der Schleifenrumpf ausgeführt wird. Beschreibe eine Bedingung oder einen Fehler, durch den sich die Schleife nicht beendet und eine Endlosschleife entsteht. Falls es so etwas nicht gibt, erkläre, wie du die Schleife ändern müsstest, um eine Endlosschleife zu erhalten.
  3. Betrachte die Prozedur aus Teil (i) des Abschnitts „Procedure“ in deinem Personalized Project Reference Sheet. Beschreibe eine Änderung, die zu einem Laufzeitfehler führt, und erkläre warum.
  4. Angenommen, du bekommst eine Prozedur 
    isEqual (value1, value2)
    , die 
    true
    zurückgibt, wenn 
    value1
    und 
    value2
    denselben Wert haben, sonst 
    false
    . Verwende die Liste, die du in deinem Personalized Project Reference Sheet angegeben hast, und beschreibe Schritt für Schritt einen Algorithmus, der mithilfe von 
    isEqual
    zählt, wie oft ein bestimmter Wert in der Liste vorkommt. Die Beschreibung muss so detailliert sein, dass jemand anderes den Code daraus schreiben kann.
