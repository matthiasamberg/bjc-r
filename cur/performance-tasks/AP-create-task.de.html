Deinen Code für die AP Create Task zeigen

Snap! bietet zwei praktische Möglichkeiten, einen Teil oder den gesamten Code zu teilen.

Script Pics erstellen

Du kannst ein PNG-Bild jedes Blocks bzw. Skripts herunterladen, indem du ihn/sie mit der rechten Maustaste anklickst (Mac: Ctrl-Klick) und „script pic…“ wählst. Die Datei landet im Downloads-Ordner deines Browsers.

Wenn du bei einem Reporter-Block mit gedrückter Shift-Taste rechtsklickst (Mac: Shift-Ctrl-Klick), enthält das Script Pic die Sprechblase mit dem Rückgabewert.

Alle Bilder auf einmal exportieren

Du kannst eine HTML-Zusammenfassung aller Blöcke deines Projekts sowie ein Bild der Bühne exportieren, indem du im Snap!-Dateimenü „Export summary…“ wählst (). Diese HTML-Datei landet ebenfalls im Downloads-Ordner. Öffne sie (im Browser), klicke die gewünschten Bilder an (Rechtsklick oder Mac: Ctrl-Klick), kopiere sie und füge sie in das Dokument ein, in dem du deinen Code beschreibst.

Du hast das ganze Jahr über Algorithmen entwickelt und Abstraktionen genutzt. Für den AP-Test musst du nur lernen, wie du darauf aufmerksam machst und sie beschreibst.

Abstraktion

Auch der letzte Schritt – das Verwenden von repeat, um die Struktur zu verdeutlichen – ist bereits eine einfache Form der Abstraktion. Viele Wiederholungen verschwinden und die Struktur wird sichtbar. Genau diese Art von Abstraktion sucht die AP Create Task jedoch nicht.

Du hast noch eine weitere Abstraktion verwendet. In Einheit 1 hast du erkannt, dass die 45°-Drehung \frac{1}{8} der gesamten 360°-Drehung ist. Darum hast du den Algorithmus so geschrieben:

Wenn du beide Achter durch Fünfen ersetzt, entsteht ein Windrad mit fünf Armen. Statt bei jeder Variante Zahlen zu ändern, hast du einen Block gebaut, der den Algorithmus enthält und in den du nur noch die gewünschte Anzahl Arme eingibst:

Genau solche Beispiele erwartet die Create Task: ein eigener Block, den du mehrfach im Projekt verwendest. Schleifen sind zwar auch Abstraktionen, aber für die Create Task sollst du eine selbst erstellte Abstraktion zeigen, nicht nur vorhandene Blöcke wie repeat.

Die Abstraktion pinwheel hilft dir dreifach:

  1. Du musst nicht mehr zwei Zahlen ändern, wenn du das Windrad anders gestalten willst – das verhindert Fehler (z. B. nur eine Zahl zu ändern).
  2. Statt überall diesen Code zu platzieren

    brauchst du nur noch
  3. Dein Code ist für andere leichter zu lesen, weil er jetzt beschreibt, was er tut: Er zeichnet ein Windrad mit acht Armen.

Immer wenn du einen Block erstellst und ihm einen sprechenden Namen gibst, baust du eine Abstraktion. Du versteckst Details und zeigst nur den Zweck. Wenn du den Pinwheel-Block verwendest, wird die Struktur deines Programms klarer. Lesende Personen müssen nicht mehr verstehen, was im Inneren von pinwheel passiert – der Name verrät, dass der Block ein Windrad mit einer bestimmten Anzahl an Armen zeichnet.

Beachte: Das komplette Programm in einen einzigen Block zu packen macht die Seite zwar kompakt, ist aber kein gutes Beispiel für Abstraktion. Die besten Beispiele für den AP-Upload sind Codeabschnitte, die du mehrfach verwendest und die dank eines guten Blocknamens verständlicher werden. Häufig kannst du den Inhalt dieses Blocks gleichzeitig als Beispiel für den Algorithmus nennen, mit dem der Block seinen Zweck erfüllt.
