Sequenzielle Sprachen

In diesem Labor lernst du andere Programmiersprachen kennen und wie du auf deinen Snap!-Kenntnissen aufbauen kannst.

Auf dieser Seite geht es um sequenzielle Programmiersprachen – und du wirst sehen, dass du sie im Grunde schon lesen kannst.

Eine Programmiersprache wird von zwei Dingen geprägt: ihrer Programmierphilosophie und ihren Notationsdetails. Die meisten modernen Sprachen sind multiparadigmatisch, aber im Kern sequenziell. Dazu gehört auch Snap!, obwohl du bereits etwas funktionale Programmierung (Verschachteln von Reportern und Prädikaten) sowie objektorientierte Ideen (Sprites als Objekte) kennengelernt hast.

Über Reporter und Befehle hast du in Einheit 1, Labor 2, Seite 4 gelernt; Prädikate in Einheit 2, Labor 1, Seite 2.

Multiparadigmatische Sprachen kombinieren verschiedene Stilrichtungen. In Snap! hast du beispielsweise gesehen:

  • Sequenzielle Programmierung – Anweisungen werden nacheinander ausgeführt.
  • Funktionale Programmierung – Berechnungen entstehen durch das Verschachteln von Reportern und Prädikaten statt durch das Hintereinanderausführen von Kommandoblöcken.
  • Objektorientierte Programmierung – Code wird um Objekte (Sprites) organisiert, die eigene Daten oder Skripte enthalten, statt um Funktionen oder Befehlsfolgen.
Es wäre schön, kurze Beispiele für jede Stilrichtung zu zeigen. --MF 17.12.24

Eine weitere Sprache desselben Typs zu lernen ist viel einfacher, als zum ersten Mal programmieren zu lernen. Hier siehst du zum Beispiel die Binärsuche aus Labor 1 dieser Einheit:

  1. Vergleicht den Snap!-Code mit dem folgenden Python-Code. Welche Anweisungen entsprechen welchen Blöcken?

Hier ist dieselbe Prozedur in Python:

def binarySearch(value, data):
    low = 0
    high = len(data)-1
    while low <= high:
        currentIndex = average(low, high)        # find the middle word
        currentItem = data[currentIndex]
        if currentItem == value:
            return True
        elif currentItem > value:                # eliminate half the list
            high = currentIndex-1
        else:
            low = currentIndex+1
    return False

Der Python-Code sieht dem Snap!-Block ziemlich ähnlich, unterscheidet sich aber in einigen Details.

JavaScript

Hier ist dieselbe Binärsuche in JavaScript:

function binarySearch(value, data) {
    var low, high, currentIndex, currentItem;
    low = 0;
    high = len(data)-1;
    while (low <= high) {
        currentIndex = average(low, high);    // find the middle word
        currentItem = data[currentIndex];
        if (currentItem == value) {
            return true;
        } else if (currentItem > value) {     // eliminate half the list
            high = currentIndex-1;
        } else {
            low = currentIndex+1;
        }
    }
    return false;
}

Auch dieser Code unterscheidet sich nur wenig, bringt aber neue Schreibweisen mit sich.

Eine weitere sequenzielle Sprache lernen

Es gibt natürlich Notationsdetails (Syntax) zu lernen. Aber weil du Snap! beherrschst, kannst du dir eine andere sequenzielle Sprache jetzt viel schneller aneignen – du kannst bereits in diesem Paradigma programmieren!

Die Syntax einer Sprache beschreibt die verwendete Notation (Schlüsselwörter, Satzzeichen usw.). Auch natürliche Sprachen haben Syntax, zum Beispiel Unterschiede zwischen Nomen und Verben oder zwischen Komma und Semikolon.

Die Semantik einer Sprache beschreibt die Bedeutung der jeweiligen Syntax. Ich bin mir nicht sicher, warum dieser Begriff hier auftaucht; er wird sonst nicht erwähnt, und ein isolierter Vokabel-Eintrag wirkt etwas seltsam. --MF, 20.12.21
