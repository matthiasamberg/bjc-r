AAP-3.B.1, AAP-3.B.5

Wie du in Einheit 1, Labor 2 gelernt hast, bedeutet prozedurale Abstraktion, ein großes Problem in kleinere Teilprobleme zu zerlegen.

Ein Block draw brick erlaubt dir, in Begriffen des eigentlichen Problems zu denken. Sobald der Block funktioniert, kannst du ihn verwenden, ohne über die Details nachzudenken – das macht den Code verständlicher.

  1. Passe draw brick so an, dass er zwei Eingaben length und width verwendet, und stelle die Pen-Einstellungen am Ende wieder zurück (z. B. set flat line ends to false).
  2. Testet euren Block mit mehreren Werten und speichert.

Eine Reihe von Ziegeln

Eine Ziegelreihe ist eine Wiederholung desselben Blocks mit Versatz:

  1. Erstellt einen Block row of bricks, der eine bestimmte Anzahl Ziegel nebeneinander zeichnet (z. B. mit einem repeat-Block).
  2. Experimentiert mit Länge und Breite, bis der Block eine saubere Reihe ergibt.

Die ganze Mauer

  1. Baut einen Block brick wall, der mehrere Reihen zeichnet und jede zweite Reihe versetzt startet, sodass die Fugen versetzt sind.
  2. Teste unterschiedliche Parameter (Ziegelbreite/-höhe, Anzahl Reihen).

Auf der AP-Prüfung kannst du gefragt werden, wie prozedurale Abstraktion den Code verbessert. Wenn du etwa eine Funktion 

calculate_tax
nutzt, musst du nicht die interne Berechnung kennen, um sie einzusetzen – genau das ist Abstraktion.
