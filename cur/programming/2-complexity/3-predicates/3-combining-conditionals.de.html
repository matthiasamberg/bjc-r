: Geschachtelte Bedingung
AAP-2.I.1

Eine geschachtelte Bedingung ist ein if- oder if else-Block innerhalb eines anderen if else-Blocks.

Betrachte die ersten beiden Beispiele oben noch einmal. Weil der Ausdruck a = b genau dann true liefert, wenn beide Werte gleich sind, ist die verschachtelte Bedingung im ersten Beispiel eigentlich überflüssig. Im zweiten Beispiel reicht es, das Prädikat im else-Teil aufzurufen. Manchmal (vor allem, wenn du kein Prädikat baust) sind geschachtelte Bedingungen jedoch sehr hilfreich.

  1. AAP-2.I Abschnitt b
    Beschreibe, was dieses Code-Segment tut.
  2. AAP-2.I Abschnitt a
    Schau dir deinen Code zum Zeichnen in drei Farben noch einmal an. Falls du noch keine geschachtelte Bedingung verwendest, kopiere den Code und erstelle eine Version, die eine geschachtelte Bedingung nutzt.
  3. AAP-2.I.4.B
    Du arbeitest an einer Wetter-App, die Outdoor-Bedingungen anhand der Temperatur (in Fahrenheit) und der Luftfeuchtigkeit (in Prozent) einordnet. Der folgende Code entscheidet, ob es „Hot and Humid“, „Hot and Dry“, „Cool and Humid“ oder „Cool and Dry“ ist.

    Angenommen, deine Sensoren messen 60 °F und 40 % Luftfeuchtigkeit. Welche Ausgabe produziert das Programm?
    Hot and Humid
    Die Temperatur 60 °F erfüllt die Bedingung > 70 nicht.
    Hot and Dry
    Die Temperatur 60 °F erfüllt die Bedingung > 70 nicht.
    Cool and Humid
    Die Temperatur ist ≤ 70, aber die Luftfeuchtigkeit 40 % erfüllt die Bedingung > 50 nicht.
    Cool and Dry
    Richtig! Temperatur ≤ 70 und Luftfeuchtigkeit ≤ 50.
