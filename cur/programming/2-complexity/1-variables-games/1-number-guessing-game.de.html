Auf dieser Seite beginnst du mit der Entwicklung eines Zahlratespiels, das eine lokale Variable nutzt, um eine geheime Zahl zu speichern.

  1. "U2L1-NumberGuessing"
  2. Beginne mit dem Bau eines neuen Befehlsblocks namens number guessing game, der den Code für das Spiel enthalten wird. Lass den Block-Editor geöffnet.

In einem Zahlratespiel versucht die spielende Person, die geheime Zahl des Computers zu erraten. Der Computer braucht eine Möglichkeit, diese geheime Zahl in einer Variablen zu speichern, um sie mit den Vermutungen zu vergleichen.

: Variable
AAP-1.A.1

Eine Variable ist wie eine beschriftete Schachtel, die immer genau einen Wert aufnehmen kann, zum Beispiel ein Wort, ein Kostüm oder eine Liste (die wiederum viele Dinge enthalten kann). Du kannst beliebig oft nachsehen, was gerade darin liegt.

Auf Einheit 1, Labor 2, Seite 2: Programme zum Sprechen bringen hast du prozedurale Abstraktion kennengelernt: Skripte, indem du sie in neue Blöcke gesteckt hast, benennen. Hier beschäftigen wir uns mit Datenabstraktion: Zahlen, Texte, Listen usw. bekommen Namen. Wenn du etwas benennst, kannst du dich darauf beziehen, ohne den exakten Wert zu kennen.
  1. Erstelle eine Skriptvariable namens secret number, in der die Zahl gespeichert wird, die erraten werden soll (Anleitung unten).

    Eine Skriptvariable erstellen

    1. Ziehe einen script variables-Block in den Skriptbereich. Du findest ihn in der Variablen-Palette.
    2. Benenne die Variable, indem du auf das orange a am Ende klickst und den gewünschten Namen eintippst. Hier soll sie secret number heißen.
    Später verwendest du die Skriptvariable, indem du sie aus dem script variables-Block herausziehst (genau wie einen Eingabewert) und an der Stelle platzierst, an der du sie im Code brauchst.
  2. AAP-1.A, AAP-3.E
    Verwende , um den Anfangswert von secret number auf eine zufällige Zahl zwischen 1 und 10 zu setzen. Im Menü des set-Blocks kannst du auswählen, welche Variable gesetzt werden soll.
    Die Variable secret number ist im set-Block nur dann auswählbar, wenn der Block hinter dem script variables-Block einschnappt.
  3. Verwende , um zu prüfen, ob deine Variable tatsächlich auf einen Zufallswert gesetzt wurde. Ziehe die Variable secret number aus dem script variables-Block und nutze sie als Eingabe für den say-Block.

Eine Skriptvariable existiert nur innerhalb des Skripts, in dem sie erstellt wurde. Außerhalb dieses Skripts kann sie weder gelesen noch verändert werden.

  1. AAP-1.A.2, AAP-3.E

    Schreibe einen Test in deinen Block, mit dem Snap! prüfen kann, ob die spielende Person die richtige Zahl erraten hat.

    1. Erstelle eine Eingabe für den Block number guessing game, die guess heißt (Block-Editor → "Add input...").
    2. Verwende und , um so lange nach einer Vermutung zu fragen, bis die spielende Person richtig liegt.
      Aktuell prüfst du nur, ob die spielende Person richtig geraten hat. In einem späteren Schritt wird der Block das ganze Spiel enthalten.
  2. Teste deinen Block, indem du ihn mit verschiedenen Vermutungen aufrufst. Funktioniert er wie erwartet?

Eine Skriptvariable ist lokal, weil sie nur in dem Skript existiert, in dem sie erstellt wurde. Eine Variable, die mit dem Knopf Make a Variable erstellt wurde, ist dagegen global und kann in jedem Skript verwendet werden.

  1. AAP-2.C

    Vergleiche eine globale mit einer lokalen Variable.

    1. Klicke in der Variablen-Palette auf Make a Variable.... Nenne sie favorite game.
    2. Wähle im Menü des Blocks die Variable favorite game aus und reihe sie in ein Skript ein, das startet, wenn die grüne Flagge angeklickt wird.
    3. Vergleiche den Unterschied zwischen den Menüs des set-Blocks aus dem vorherigen Schritt und des Blocks show variable.
    4. Notiere dir, was der wesentliche Unterschied zwischen einer Skriptvariablen und einer globalen Variablen ist.
: Lokale Variable
AAP-1.A.4

Eine lokale Variable kann nur innerhalb des Skripts (oder Blocks) verwendet werden, in dem sie erstellt wurde.

: Globale Variable
AAP-1.A.5

Eine globale Variable kann in jedem Skript des Projekts verwendet werden.

AAP-1.A, AAP-1.B

Variablen wirken vielleicht abstrakt, aber sie sind ein grundlegendes Werkzeug, mit dem Computer Werte speichern und verändern können.

  1. Beantworte die folgenden Fragen, um dein Verständnis zu prüfen.

    Welchen Wert gibt dieses Skript zurück?
    3
    Richtig! Die Variable b erhält den Wert der Variable a (der 3 ist, wenn 
    b ← a
    ausgeführt wird).
    4
    Die Variable b bekommt den Wert der Variable a (der 3 beträgt, wenn 
    b ← a
    ausgeführt wird); b merkt sich nicht, dass die 3 von a stammt, und ändert sich deshalb nicht, wenn sich a später ändert.
    a
    Die Variable b wurde auf den Wert der Variable a gesetzt, nicht auf den Buchstaben „a“ selbst.
    b
    Das Skript zeigt den Wert der Variable b an, nicht den Buchstaben „b“.
  2. Welchen Wert gibt dieses Skript zurück?
    -5
    Richtig!
    3
    Der Wert der Variable m ist in diesem Skript immer größer als der Wert der Variable k, daher ergibt k minus m eine negative Zahl.
    -4
    Der Wert der Variable m ist zu Beginn 9, aber er hat sich geändert, bevor er von k abgezogen wird.
    5
    Der letzte set-Block setzt den Wert der Variable k auf den Wert k – m, nicht auf m – k.
