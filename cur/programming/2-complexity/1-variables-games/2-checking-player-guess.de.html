Die Vermutung prüfen

Menschen lassen sich von der Box „Auf der AP-Prüfung“ verwirren und versuchen, den dortigen Code nachzubauen. --MF, 1.8.23

Auf dieser Seite entwickelst du dein Zahlratespiel weiter, sodass Spielende so lange raten können, bis sie richtig liegen.

Der Computer soll die Spielenden immer wieder um eine neue Vermutung bitten, bis sie richtig geraten haben. Dafür verwendest du den Block . repeat until ist eine Schleife (genau wie repeat, forever und for), aber auch ein Bedingungsblock (wie if und if else). Er wiederholt sich, bis eine bestimmte Bedingung erfüllt ist. In diesem Programm soll der Code so lange wiederholt werden, bis die Antwort der spielenden Person gleich der geheimen Zahl ist.

Schleifen hast du auf Einheit 1, Labor 3, Seite 6: Schleifen mit Zähler kennengelernt.

Bedingungen sind dir bereits von Einheit 1, Labor 2, Seite 5: Mehr Abwechslung für Gossip und von Einheit 1, Labor 5, Seite 2: Figur folgt einer Figur bekannt.

  1. Nutze repeat until, um die spielende Person () so lange nach der geheimen Zahl fragen zu lassen, bis ihre dem secret number entspricht.

    • Die Blöcke ask und answer gehören zusammen. Wenn du mit ask eine Frage stellst, liefert answer die Antwort der Nutzerin oder des Nutzers.

Der repeat until-Block prüft die Bedingung am Ende jeder Wiederholung. Wenn die Bedingung true ist, hört die Schleife auf, ansonsten wird der Inhalt noch einmal ausgeführt.

AAP-3.B.1

Auf der AP-Prüfung ist das report-Kommando ein return-Befehl. Ein zweiter return im selben Block ist erlaubt, solange der erste nicht bereits ausgeführt wurde.

  1. Teste dein Verständnis.

    Welche Eingaben für mystery function führen zum Ergebnis „finished“?
    Beliebige ganze Zahl
    Richtig! Egal welche ganze Zahl eingegeben wird, der report-Block wird irgendwann ausgeführt. (Das gilt nicht für jedes Verfahren, aber hier schon.)
    Nur ganze Zahlen größer oder gleich 5
    Schreibe einige Beispiele auf, bei denen die Eingabe kleiner als 5 ist. Wenn die Eingabe zum Beispiel 3 ist, sagt der Sprite 3, die Eingabe wird zu 4, der Sprite sagt 4, die Eingabe wird zu 5, die repeat until-Schleife stoppt und das Verfahren meldet „finished“.
    Nur ganze Zahlen kleiner als 5
    Der report-Befehl steht nicht innerhalb des if-Blocks.
    Nur 5
    Der repeat until-Block prüft den Wert von input bei jedem Durchlauf, nicht nur am Anfang.
  2. Was sagt der Sprite, wenn du dieselbe Funktion mit der Eingabe 1 aufrufst?
    Am Ende sagt der Sprite 4, aber zuvor werden noch andere Zahlen angezeigt.
    Genau! Das Skript sagt 1, verändert die Eingabe zu 2, sagt 2, verändert die Eingabe zu 3 und so weiter, bis es 5 sagt.
    Vergiss nicht, dass die Schleife so lange weiterläuft, bis die Eingabe 5 ist.
    Der Wert der Eingabe ändert sich innerhalb der Schleife.
