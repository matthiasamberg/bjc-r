Eine mathematische Bibliothek erstellen

In diesem Projekt baust du eine Bibliothek (eine Sammlung) von Mathe-Werkzeugen auf, die auf den Blöcken aus Labor 3, Seite 2 aufsetzt.
AAP-2.M.3, AAP-3.D.3

Bibliotheken vereinfachen die Entwicklung neuer Programme. Wenn du Prozeduren nutzt, von denen du weißt, dass sie funktionieren, reduzierst du Programmieraufwand, potenzielle Fehlerquellen und den Testumfang.

: Softwarebibliothek
AAP-3.D.1

Eine Softwarebibliothek ist eine Sammlung von Prozeduren, die in Programmen verwendet werden können.

  1. Exportiere die mathematischen Blöcke aus deinem Projekt U2L3-Predicates.

    Blöcke exportieren

    1. Öffne das Snap!-Projekt, aus dem du exportieren möchtest (hier: U2L3-Predicates).
    2. Wähle „Export blocks...“ im Snap!-Dateimenü ().
    3. Markiere nur die Blöcke, die du exportieren willst (hier z. B. , , und ).
      Du kannst nicht benötigte Blöcke abwählen oder das Kontextmenü öffnen, „none“ wählen und danach die gewünschten Blöcke anklicken.
    4. Snap!-Bibliotheken und -Projekte werden beide als XML-Dateien exportiert.
      Klicke „OK“. Eine XML-Datei wird heruntergeladen.
  2. Importiere diese Blöcke in dein Projekt U2L4-MathLibrary, teste sie, behebe Probleme und speichere.

    Blöcke importieren

    1. Öffne das Snap!-Projekt, in das du importieren willst (hier: U2L4-MathLibrary).
    2. Ziehe die XML-Datei aus deinem Download-Ordner in das Snap!-Fenster.
      Alternativ kannst du im Dateimenü „Import...“ wählen, die XML-Datei suchen und „Open“ anklicken.
    3. Teste die importierten Blöcke:
      1. Du findest sie am Ende der jeweiligen Palette.
      2. Klicke jeden Block an, um sicherzugehen, dass er funktioniert.
Du erstellst gerade eine Mathebibliothek. Snap! bietet zahlreiche Bibliotheken, zum Beispiel für die Arbeit mit Textbausteinen. Bibliotheken lassen sich im Dateimenü über „Libraries“ aktivieren.
  1. Erstelle die folgenden Blöcke und teste sie. Beschreibe kurz, was sie tun und welche Eingaben sie erwarten.
    1. divisors of: Liefert eine Liste aller Teiler einer positiven ganzen Zahl.
    2. has exactly two divisors?: Meldet true, wenn die Eingabe genau zwei Teiler hat.
    3. between?: Nutzt deine exportierten Blöcke, um zu prüfen, ob eine Zahl zwischen zwei Grenzen liegt.
  2. Schreibe eine Kurzbeschreibung (z. B. in einer Textbox in Snap!), wie man jeden deiner Blöcke verwendet. Das ist der Beginn der Dokumentation für deine Bibliothek.
  3. AAP-2.M
    Teste deine Blöcke ausführlich: Welche Zahlen haben genau zwei Teiler?
    • Welche Arten von Zahlen könnten mehr als zwei Teiler haben?
    • Welche Zahlen haben genau zwei Teiler?

Du hast nun eine kleine Bibliothek erstellt. Es ist sinnvoll, eine Anleitung zur Nutzung der Funktionen zu schreiben (zum Beispiel: divisors of erwartet eine positive ganze Zahl und liefert eine Liste von Zahlen). Diese Anleitung ist die Application Program Interface deiner Bibliothek.

: Application Program Interface (API)
AAP-3.D.4, AAP-3.D.5

Eine Application Program Interface (API) dokumentiert, was eine Programmiererin über eine Bibliothek wissen muss: Sie beschreibt Zweck, Eingaben und Ausgaben jeder Prozedur (aber nicht den Algorithmus).

Was ist eine Web-API?

Eine häufige Form einer API ist eine Web-API, bei der die Bibliothek auf einem anderen Rechner liegt. Beispiel: Die Google-Maps-API erklärt, wie du eine Google-Karte auf deiner Website einbindest.
