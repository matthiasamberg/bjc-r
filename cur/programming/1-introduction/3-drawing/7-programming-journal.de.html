Ein Programmierjournal führen

Auf dieser Seite erfährst du, wie dir ein Programmierjournal beim Arbeiten hilft.
Notiere dir täglich zum Beispiel:
  • Programmierkonzepte, die dir Schwierigkeiten bereitet haben.
  • Fragen, die du noch stellen möchtest.
  • Ideen für neue Projekte oder Verbesserungen an bestehenden Projekten.
  • Probleme in deinem Code:
    • Welches Symptom ist aufgetreten?
    • Was war die Ursache?
    • Welche Änderung hat das Problem behoben?
    • Mit welchen Techniken hast du das Problem untersucht?
  • Wie hast du deinen Code organisiert?
    • Hast du neue Prozeduren definiert?
      • Nehmen sie Eingaben? Wenn ja: Welche Art von Eingaben (ihr Domain)?
      • Liefern sie einen Wert? Wenn ja: Welche Art von Wert (ihr Range)?
    • Gab es Stellen, an denen du stattdessen eine Prozedur hättest definieren können, statt Code zu kopieren?
  • Wenn es passt, füge Bilder der Bühne oder wichtiger Skripte hinzu.

Du musst nichts aus dem Curriculum abschreiben; alles findest du im Index.
Das Journal ist für dich, nicht für die Lehrkraft. Es wird nicht bewertet – es hält fest, was du erlebt hast, nicht nur die Inhalte des Curriculums. Deshalb wird jedes Journal anders aussehen.
