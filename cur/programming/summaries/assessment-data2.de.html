2.1.3
Welche Aussage beschreibt am besten eine 3D-Computergrafik?
Sie erfordert Virtual-Reality-Brillen.
Ein 3D-Modell kann auch ohne VR-Brille betrachtet werden.
Sie besitzt Breite, Höhe und Tiefe.
Richtig!
Sie zeigt Bewegungen für Animationen.

2.1.3
Welche zwei Aktivitäten passen am besten zum Konzept „Pair Programming“? (Wähle zwei.)
Eine Person schreibt am Computer, die andere liest Dokumentation mit Bezug zum Projekt.
Pair Programming bedeutet aktives Arbeiten am gleichen Code.
Eine Person gibt Code ein, die andere beobachtet, erkennt Fehler und macht Vorschläge.
Richtig!
Beide schreiben unabhängig voneinander an verschiedenen Modulen.
Das ist Arbeitsteilung, kein Pair Programming.
Die Lernenden wechseln sich am Computer ab, halten dabei ständig miteinander Rücksprache.
Richtig! Rollenwechsel gehört dazu.
2.1.3
Worin unterscheidet sich die Denkweise beim Pair Programming von Einzelarbeit?
Die Person, die tippt, braucht keine Kenntnisse.
Falsch – beide Benötigen Verständnis.
Die beobachtende Person bleibt passiv.
Sie spürt Fehler auf und denkt mit.
Beide müssen gemeinsam planen und kommunizieren.
Richtig!
Das Ziel ist, doppelt so viel Code in der halben Zeit zu schreiben.
Pair Programming konzentriert sich auf Qualität, nicht auf reine Geschwindigkeit.

Labor 2: Listen erstellen

2.2.1
Welche Darstellung ist eine Liste?
Ein einzelnes Wort.
Eine Liste enthält mehrere Einträge (auch wenn es nur einer sein kann, die Darstellung ist charakteristisch).
Ein Zahlenpaar im Format (5,7).
Das ist ein geordnetes Paar, kein Listensymbol in Snap!.
Richtig!
Eine Zeichenkette mit Kommas (z. B. „rot, blau, gelb“).
Das ist ein String; in Snap! wäre das eine Textvariable.
2.2.1
Betrachte folgenden Code:

Welche Ausgabe erhältst du?

Achte darauf, dass ein neues Element eingefügt wird.

Richtig!
{pizza, burger, taco, sushi}
