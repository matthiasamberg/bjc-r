Labor 1: Spiele

2.1.1: Variable  
AAP-1.A.1

Eine Variable ist wie eine Box, die immer genau einen Wert aufnehmen kann, z. B. ein Wort, ein Kostüm oder eine Liste (die wiederum viele Dinge enthalten kann). Du kannst beliebig oft nachsehen, was aktuell darin liegt.

In Einheit 1, Labor 2, Seite 2 hast du prozedurale Abstraktion kennengelernt: Skripte erhalten Namen, indem du sie in neue Blöcke packst. Hier geht es um Datenabstraktion: Zahlen, Texte, Listen usw. bekommen Namen. Wenn du etwas benennst, kannst du dich darauf beziehen, ohne den genauen Wert zu kennen.
2.1.1: Lokale Variable  

Eine lokale Variable kann nur dort gesetzt oder verwendet werden, wo sie definiert wurde. Dazu zählen Eingaben von Prozeduren sowie Variablen, die über for oder script variables erstellt werden.

Warum wird „Boolesch“ großgeschrieben?

Der Begriff geht auf George Boole zurück, der die Boolesche Algebra entwickelte. Eigennamen werden großgeschrieben.

Labor 3: Entscheidungen treffen

2.3.1: Definitionsbereich und Wertebereich  
  • Der Eingabetyp (Definitionsbereich) einer Funktion beschreibt, welche Daten als Eingabe akzeptiert werden.
  • Der Ausgabetyp (Wertebereich) beschreibt, welche Daten als Ergebnis zurückgegeben werden.
2.3.1: Sequencing, Selection, Iteration  
Selection: AAP-2.G.1; Sequencing/Selection/Iteration: AAP-2.A.4

Selection bedeutet zu entscheiden, welcher Teil eines Algorithmus ausgeführt wird (if, if else).

Jeder Algorithmus lässt sich aus Sequencing (Schritte der Reihe nach), Selection (Entscheiden) und Iteration (Wiederholen) zusammensetzen.

2.3.3
