Internet-Abstraktionen und offene Protokolle

Auf dieser Seite lernst du die Kommunikationsstandards kennen, die im Internet verwendet werden, und wie sie zusammenarbeiten.

: Protocol (Protokoll), IP Address (IP-Adresse), Packet (Paket) und Packet Switching (Paketvermittlung)
CSN-1.B.3, CSN-1.C.1
  • Ein protocol (Protokoll) ist eine Sammlung von Regeln, die das Verhalten eines Systems festlegen.
  • Eine IP address (IP-Adresse) ist eine eindeutige Nummer, die jedem Gerät in einem Computernetzwerk zugewiesen wird.
  • Ein packet (Paket) ist ein kleiner Block beliebiger Daten (Text, Zahlen, Listen usw.) samt Metadaten (Informationen über diese Daten), der als Datenstrom durch das Internet gesendet wird.
  • Packet switching (Paketvermittlung) bedeutet, dass das Internet Informationen in kurzen Burst-Paketen versendet, nicht als lange, kontinuierliche Zeichenketten.

Milliarden von Geräten sind mit dem Internet verbunden und es gibt Hunderte unterschiedlicher Gerätetypen: Laptops, Tablets, Telefone, smarte Kühlschränke, mobile Kartenleser und vieles mehr. Wie finden und verstehen sich all diese Geräte? Protokolle (Kommunikationsstandards) sorgen dafür, dass die Vielfalt der Geräte reibungslos miteinander interagiert.

Jedes Gerät im Internet hat eine eigene Internet Protocol (IP) address (oder mehrere, wenn es ein Router ist) – vergleichbar mit einer Post- oder E-Mail-Adresse. Das Internet Protocol legt fest, wie ein Router mit einer Anfrage nach einer anderen IP-Adresse umgeht. Jeder Router kennt die Struktur seiner Umgebung im Internet und weiß, wohin eine Nachricht als nächstes gesendet werden muss, damit sie ihrem Ziel näher kommt. Dass Router nicht alle Adressen des gesamten Internets kennen müssen, verbessert die Skalierbarkeit.

Wenn du Daten über das Internet streamst, wird der Strom in Pakete aufgeteilt, die IP einzeln versendet. Dieser Prozess macht das Internet zu einem Netzwerk mit Paketvermittlung.

CSN-1.C.2, CSN-1.C.3

Das Internet ist ziemlich zuverlässig, doch hin und wieder geht ein Paket verloren, und die beteiligten Geräte müssen solche Fehler tolerieren. Eine Möglichkeit besteht darin, den Verlust zu ignorieren (wenn ein Videoframe fehlt, ist das kaum schlimm). Eine andere Möglichkeit (genannt TCP für Transmission Control Protocol) besteht darin, Pakete so lange zu senden, bis ihr korrekter Empfang bestätigt wird. Bei Anwendungen, die TCP nutzen, übernimmt TCP selbst die Aufteilung der Daten in Pakete. Weil Pakete unterschiedliche Wege nehmen können, kommen sie möglicherweise in anderer Reihenfolge an; und trotz der Redundanz des Internets kann es sein, dass manche Pakete gar nicht ankommen. TCP garantiert eine zuverlässige Datenübertragung, indem es nachverfolgt, welche Pakete erfolgreich empfangen wurden, verlorene oder beschädigte Pakete erneut sendet und die Reihenfolge für das Zusammensetzen der Daten am Ziel vorgibt.

: TCP/IP

TCP/IP ist ein Protokollpaar, das zwei Abstraktionsebenen bereitstellt:

  • IP (Internet Protocol) lässt deinen Computer so tun, als hätte er eine direkte Verbindung zu einem anderen Computer. Die Router an jedem Verbindungspunkt im Internet verwenden IP, um Pakete von einer IP-Adresse zur nächsten zu übertragen.
  • TCP (Transmission Control Protocol) lässt deinen Computer so tun, als hätte er eine zuverlässige Verbindung zum anderen Computer. Die Computer (einschließlich Server) an den beiden Endpunkten einer Kommunikation verwenden TCP, das sicherstellt, dass Pakete erfolgreich ankommen.

Hierarchie der Internet-Abstraktionen

CSN-1.D.2 (unter „Application Layer Protocols“)
CSN-1.C-4

Beispiele für Protokolle je Ebene:

Application Layer: Browser verwenden HTTP (HyperText Transfer Protocol), um HTML-Anweisungen für die Darstellung von Webseiten zu interpretieren. DNS (Domain Name System) wandelt benutzerfreundliche Hostnamen (wie edc.org oder berkeley.edu) in IP-Adressen um. Deine E-Mail-Anwendung nutzt möglicherweise SMTP (Simple Mail Transfer Protocol) zum Senden und IMAP (Internet Message Access Protocol) zum Lesen von E-Mails.

Transport Layer: TCP (Transmission Control Protocol) simuliert eine zuverlässige, dauerhafte Verbindung zwischen zwei Computern, indem Daten erst angezeigt werden, wenn alle Pakete angekommen sind. Wenn Geschwindigkeit wichtiger als Genauigkeit ist, zum Beispiel bei Echtzeit-Video, kommt UDP (User Datagram Protocol) zum Einsatz.

Internet Layer: Jedes Gerät im Internet benötigt eine IP-Adresse, damit andere Geräte es finden können. Router verwenden Protokolle der Internet-Schicht, um Netzüberlastungen zu erkennen und zu umgehen.

Network Interface Hardware: Du verbindest dich vielleicht mit einem Ethernet-Kabel oder über eine WLAN-Antenne im Gehäuse deines Computers mit dem Internet. Beide verbinden den Computer mit einem lokalen Netzrouter, der dann den Kontakt zum Internetanbieter (ISP) herstellt. Mobiltelefone nutzen eine länger reichende Mobilfunk-Verbindung zu ihrem Netzbetreiber.

Es gibt sehr viele Protokolle! Das Internet wurde mit mehreren Ebenen von Abstraktion entwickelt, die die Protokolle danach ordnen, welchen Teil des Kommunikationsprozesses sie unterstützen. Diese Abstraktionshierarchie beherrscht die Komplexität des Internets, indem sie die Details der unteren Ebenen verbirgt:

  • Application Layer Protocols (z. B. HTTP) bilden die höchste Abstraktionsebene, weil sie steuern, wie Daten interpretiert und angezeigt werden. Diese Protokolle geben den Bits Bedeutung, die von tieferen Ebenen gesendet werden; Client- und Server-Computer müssen sich darüber einig sein, und die Anwendungsprotokolle stellen diese Einigung her.
  • Transport Layer Protocols (z. B. TCP) verwalten die Aufteilung einer Nachricht in Pakete, die von unteren Ebenen übertragen werden, sowie die Zusammensetzung der Nachricht aus den Paketen bei der Ankunft.
  • Internet Layer Protocols (z. B. IP) organisieren die Pfade, auf denen Datenpakete über Netzwerke reisen. Diese Protokolle vermitteln die Abstraktion eines großen Netzwerks, obwohl auf physischer Ebene zahlreiche Teilnetze existieren.
  • Network Interface Hardware (mit Link-Layer-Protokollen wie WLAN) verwaltet die Verbindung zwischen einem Internetgerät und seinem lokalen Netzwerk. Diese lokalen Protokolle sind die am wenigsten abstrakten, weil sie direkt mit deiner physischen Hardware arbeiten.
CSN-1.B.4 (Absatz über „open standards“)
All diese Standards sind offen: Jeder kann ein Protokoll nachschlagen und damit neue Hardware oder Software entwickeln, ohne jemanden um Erlaubnis bitten zu müssen. Das Internet ist wahrscheinlich das größte und komplexeste Artefakt der Menschheitsgeschichte und es basiert auf Zusammenarbeit. Trotz einzelner Versuche von Regierungen, das Netz zu zensieren, überwiegt der kooperative Geist.
Warum sind offene Standards wichtig?
  • Dein T-Mobile-Handy kann mit dem Verizon-Handy deiner Freundin sprechen.
  • Du kannst einer Person in einem Land eine E-Mail senden, das als feindlich gegenüber deinem Land gilt (zum Beispiel von den USA nach Iran).
  • Eine Ingenieurin bei Microsoft kann eine Webseite bei Apple lesen, obwohl ihre Unternehmen Wettbewerber sind.

Vor dem Internet gab es mehrere verschiedene Netzwerkprotokolle, die als Geheimnisse bestimmter Hersteller galten. Hattest du also einen Computer oder Router einer bestimmten Marke, konnte er nur mit Geräten derselben Marke kommunizieren.
