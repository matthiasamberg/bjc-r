Eine API (Application Programming Interface) ist nicht nur fürs Web relevant; sie entspricht dem Abstraktionsgedanken aus Woche 1. Sie definiert, wie ein Service/Programm/Block genutzt wird. Man kann beeindruckende Anwendungen bauen, indem man Web-APIs miteinander „verklebt“.

Letzte Woche hast du dich gefragt, ob es einen besseren Weg gibt, Daten von Webdiensten (z. B. Wetter für eine PLZ) zu holen. Den gibt es – aber pädagogisch wollten wir dir erst den „schlam­mi­gen“ Weg zeigen, bevor du drüberfliegst.

Wie du Bibliotheksblöcke in Snap! aufrufst (map, substring …), kannst du auch Programme im Web mit Parametern versorgen, damit sie

  • etwas ausführen (Side Effect, wie ein Befehlsblock)
  • einen Wert liefern (Reporterblock)
  • oder beides (z. B. Reporter mit Rückmeldung, ob der Effekt geklappt hat – wie ein Butler: „Jeeves, schalt den TV ein.“ – „Sorry, Fernbedienung nicht gefunden.“)

Wir starten mit APIs, die einfache, lesbare Werte liefern – genau jenes API vom letzten Lab.

  1. Öffne einen neuen Tab und nutze den Link http://ipv4bot.whatismyipaddress.com/. Er liefert deine IP-Adresse (wie letzte Woche).
  2. Viele APIs benötigen Eingaben. Dieses hier gibt dir zur IP die PLZ zurück: http://ipinfo.io/207.192.252.171/postal.
  3. Noch eins? Zufälliges englisches Wort: http://randomword.setgetgo.com/get.php.
  4. Starte ein neues Snap!-Projekt, gib deinem Sprite ein Kostüm und lass es beim grünen Flaggenklick sagen: „Klick mich für ein zufälliges Wort!“. Beim Anklicken soll es das Wort aus dem obigen API anzeigen und anbieten, noch eins zu liefern.
  1. Wie könntest du das API „stressen“, um herauszufinden, wie viele Wörter in der Datenbank stecken?
  2. Nutze das Zufallswort-API in deiner „Guess a Word“-Hausaufgabe aus BJC.2x, statt Wörter aus einer gespeicherten Liste zu nehmen.

Lab-Quiz

Hat das Zufallswort-API 100 Wörter und du darfst eins pro Sekunde anfordern: Wie lange dauert es garantiert, bis du alle Wörter gesehen hast?
