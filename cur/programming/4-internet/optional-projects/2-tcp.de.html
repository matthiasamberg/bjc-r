Computer, Server und Router sind meist stabil, aber gelegentlich geht ein Paket verloren – Geräte im Netz müssen damit umgehen. Eine Möglichkeit: Es ist egal (z. B. verlorene Videoframe). Die andere (TCP): Pakete werden so lange gesendet, bis sie bestätigt sind. TCP (Transmission Control Protocol) garantiert zuverlässige Übertragung, indem es verfolgt, welche Pakete angekommen sind, fehlende oder beschädigte erneut sendet und die Reihenfolge fürs Zusammensetzen vorgibt.

TCP/IP ist ein Protokollpaar, das eine Abstraktion bereitstellt. IP lässt deinen Rechner so tun, als hätte er eine direkte Verbindung zu einem anderen. TCP lässt ihn glauben, diese Verbindung sei zuverlässig.

Das TCP/IP-Ende-zu-Ende-Design ist eine Abstraktion:

  • Endgeräte (inkl. Server) führen TCP aus und sorgen für Zuverlässigkeit.
  • Router auf allen Verbindungen führen IP aus und leiten Pakete weiter.

Router wissen nichts über den Inhalt; sie übertragen nur. Nur Sender und Empfänger kümmern sich um die Bedeutung. Neue Anwendungen erfordern somit keine neuen Router – für sie sind alles nur Bits.

TCP ergänzt jedes Paket um Infos, damit der Empfänger verfolgen kann, was angekommen ist, fehlende Pakete nachfordern und alles korrekt zusammensetzen. In dieser Simulation kommt ein Paket entweder korrekt (evtl. in falscher Reihenfolge) an oder gar nicht. Im echten Internet können Pakete fehlerhaft eintreffen, daher prüft TCP auch die Inhalte und fordert fehlerhafte erneut an.
