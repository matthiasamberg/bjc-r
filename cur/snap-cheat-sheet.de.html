In Snap! zeigt die Form eines Eingabefelds, welcher Datentyp erwartet wird. Beim Block ist das erste Feld rund (Zahl) und das zweite Feld hat ein Listensymbol:

Über das Dropdown-Menü kannst du den ersten, letzten oder einen zufälligen Eintrag wählen, ohne eine Zahl einzugeben.

Auf den zweiten Blick ist klar: Dieser Block ersetzt den Eintrag an Position i durch den Eintrag an Position j derselben Liste.

Beispiel:

In Snap! solltest du Variablen nicht einfach list nennen, weil es bereits einen gleichnamigen Block gibt. Verwende beschreibende Namen, zum Beispiel:

Leere Kommando-Definition

Blöcke, die etwas tun, heißen in Snap! Kommandos. Sie haben die typischen rechteckigen Puzzleformen, damit sie zu Skripten zusammengesteckt werden können.

Beim Erstellen eines Blocks legst du Namen, Eingaben und die Palette (Farbe) fest und baust dann das Skript, das beim Anklicken ausgeführt wird. Beispiel:

Weitere Hinweise zum Blockbau findest du in diesem Spickzettel.

{
    <instructions>
    RETURN expression
}
Leere Reporter-Definition

Blöcke, die einen Wert melden, heißen Reporter. Sie sind abgerundet und lassen sich anklicken oder als Eingabe verwenden.

Beim Erstellen eines Reporters ist automatisch ein melde-Block enthalten; dessen Eingabe definiert den Rückgabewert.
