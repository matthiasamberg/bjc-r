Best Practices für HTML in BJC4NYC

Siehe auch: EDC-Template und Style Guide.

Diese Seite hat Mary zusammengestellt, nachdem sie einen Vormittag lang HTML-Fehler gesucht hatte. Es gibt nicht nur eine „richtige“ Art, verschachtelte Tags einzurücken, aber wir sollten gemeinsam lesbaren, konsistenten Code schreiben. Unten steht die Methode, die ich nutze, als Empfehlung. Bitte lesen (und gern Feedback geben) – und haltet euch daran. Danke! :) Mary

  1. Schreibe aufgeräumten Code mit gut lesbarem Format:
    • Nutze Zeilenumbrüche, um Codeabschnitte zu trennen – nicht Absätze;
    • Verwende Tabs/Einrückungen, um verschachtelte Tags sichtbar zu machen (Beispiel: Quellcode dieser Seite oder dieses einfache Beispiel);
    • Denk beim Formatieren daran, dass Teammitglieder den Code lesen und debuggen müssen. :)
  2. Vor dem Commit auf valide HTML-Struktur prüfen:
    • Stelle sicher, dass alle geöffneten Tags geschlossen werden (außer selbstschließende wie <img /> oder <br />); bei starker Verschachtelung helfen saubere Einrückungen enorm.
    • Teste die Seite in mehreren Browsern (bei reinen Textänderungen optional, bei Layout-Änderungen Pflicht).
  3. Style Guide befolgen:
    • Kopiere HTML/CSS direkt aus dem Style Guide oder überprüften Beispielen (andere Seiten sind weniger sicher, da sie angepasst worden sein könnten).
    • Nutze vorhandene Komponenten aus dem Style Guide, statt defekte Elemente zu „ersetzen“. Wenn etwas daraus nicht funktioniert, behebe es, melde ein Issue auf GitHub oder frag nach Hilfe. Bitte keine lokalen Workarounds bauen, wenn eigentlich etwas Globales repariert werden sollte.
      Beispiele:
      • Beispiel 1: <div><span><strong>Sasha:</strong></span> Ich sehe noch eine Möglichkeit...</div>
        Im Dialog-Pattern (<div class="dialogue">) erzeugt das innere <span> den Fettdruck für den Namen. Wenn das nicht passiert, stimmt etwas im HTML nicht. Bitte das Problem beheben oder melden, statt den Namen manuell fett zu setzen.
      • Beispiel 2: <div class="sidenote">&nbsp;&nbsp; ... &nbsp;Was macht Tonys <code>map</code>-Block genau?</div>
        Wenn du viele geschützte Leerzeichen brauchst, um ein Sticky-Note-Layout zu erzwingen, deutet das auf einen Fehler hin. Bitte repariere ihn, dokumentiere ihn oder gib ihn weiter.
  4. Dateinamen serverfreundlich halten: keine Sonderzeichen (z. B. „?“), keine Großbuchstaben und keine Leerzeichen (stattdessen „-“). Der Server ist case-sensitiv, daher müssen Referenzen exakt zum Dateinamen passen.
    • z. B. statt „SpaceKeyPressed? 214 × 28 pixels.png“ → „space-key-pressed-214x28.png“ (kleines x statt Multiplikationszeichen)
