Didaktische Designprinzipien

  • Weniger Text und Ablenkung, kürzere Seiten, klare Projektorientierung.
  • Testet Seiten vollständig; jede Anweisung muss Lernende zuverlässig zum Ziel führen.
  • CS-Konzepte nur einsetzen, wenn sie sinnvoll sind; nichts erzwingen.
  • Generalisierungen nur einbauen, wenn ein konkreter Bedarf besteht.

Schreibkonventionen

  • Keine Berkeley-/CS10-spezifischen Anspielungen; Du-/Ihr-Form verwenden.
  • „fun“ und „let’s“ (außer in Dialogen) vermeiden; Stil zwischen locker und formal.
  • „Block“ = ein Block; gefüllte Reporter sind Expressions, Kommandos Instructions, mehrere Blöcke = Script/Program.
  • Listen haben Items; Reporter reporten Werte.
  • Snap!-UI-Texte in Anführungszeichen; `snap` verwenden.

HTML-Richtlinien

  • Neue Begriffe per `` hervorheben, Betonung via ``.
  • Jeder Textblock gehört in `

    ` oder `

    `; Listen immer eingerahmt ablegen.
  • Grafiken nicht in `div` packen, sondern Klassen wie `imageRight`, `imageLeft`, `indent` nutzen.
  • Keine `height`/`width`-Attribute im ``; Bilder vorab korrekt skalieren.
  • Mit Tabs/4 Leerzeichen einrücken, damit der Code lesbar bleibt.

Datei- und Blockbenennung

Der Server ist case-sensitive – Referenzen müssen exakt passen.
  • Bilder: lowercase, keine Leerzeichen/Sonderzeichen, Ablage in passender Unit unter `/img` (Blockshots → `/img/blocks`).
  • Snap!-Projekte: `UxLy-TitleCase.xml`, gleiche Bezeichnung in Snap! selbst; Datei in `/prog//`.
  • Lab-Seiten: `labpage-title.html`, alles lowercase mit Bindestrichen.
  • TG-Seiten: gleiches Schema, Start mit Labnummer.
  • Blöcke: sprechende Namen, keine Leerzeichen, Snap!-Palette passend wählen.

KaTeX

Dollar-Delimiters bitte nicht mehr nutzen. Wir setzen auf KaTeX.

  • Inline: `\frac{a}{b}`; Block: `
    ...
    `.
  • Komplexe Formeln in `\text{}` beschriften, damit Screenreader sauber vorlesen.

CSS-Komponenten & Nummerierung

Die wichtigsten Klassen:

  • div.class="takeNote" – gelbe Hinweisbox.
  • div.class="forYouToDo" id="first" – FYTD; weitere FYTDs mit `start="4"` etc.
  • div.class="sidenote" / sidenoteBig – Zusatzinfos (20 % / 40 %).
  • div.class="endnote" – optionale Lektüre oder Hints (mit `collapse`-Pattern).
  • div.class="dialogue" – Dialoge; jede Zeile eigenes `
    `, Namen im ``.
  • div.class="ifTime" – „If There Is Time“, Nummerierung an FYTD fortsetzen.
  • div.class="takeItFurther" – „Take It Further“, Großbuchstaben (A, B, …) nutzen; Unterpunkte über CSS (lower-roman).
  • div.class="sideHOM" / sideHOMbig – große orange Boxen.
  • div.class="vocab" – Vokabelkästen (Begriff fett im Titel, im Text kursiv).
  • div.class="todo" – sichtbare TODOs, die vor Release entfernt werden.

Code, Icons & Bilder

  • Code-Fragmente in ``; längere Blöcke mit `
    ...
    `.
  • Icons (z. B. Tough Stuff) aus `/img/icons/` einbinden; Alt-/Title-Text setzen.
  • Bilder von Blöcken via Smart Pics; Breite max. Inhaltsspalte, sonst `imageRight` einsetzen.
  • Animationen: GIF oder Video mit klarer Beschriftung; alt/title nötig.

Topic-Icons & Snap!-XML

  • Topic-Seiten nutzen `topic_icons.css` – wähle Icon-Klassen gemäß Referenz.
  • Snap!-XML-Dateien im `/prog`-Baum halten; Metadaten (Titel, Autoren) in Snap! pflegen.
  • Blöcke/Assets, die in mehreren Units vorkommen, zentral ablegen und verlinken.

Links, Präsentationen & Anchors

  • Projekte über `#present:username=...&project=...`-URLs einbetten.
  • Schalter: `#present&no_run`, `#editMode`, `#cloud` (Projekt aus Cloud laden).
  • Anker: `...` + `` (oder `id`).
  • Wenn nur ein Abschnitt verlinkt wird, wähle sprechende IDs (z. B. `#step-3`).
